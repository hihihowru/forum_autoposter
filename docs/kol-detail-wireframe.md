## 響應式設計

### 桌面版 (≥1200px)
- 3列佈局：基本資料、發文設定、統計概覽
- 人設設定：2列佈局
- 發文歷史：全寬表格
- 互動分析：全寬圖表 + 3列統計卡片

### 平板版 (768px - 1199px)
- 2列佈局：基本資料+發文設定、統計概覽
- 人設設定：2列佈局
- 發文歷史：全寬表格
- 互動分析：全寬圖表 + 2列統計卡片

### 手機版 (<768px)
- 1列佈局：所有卡片垂直堆疊
- 人設設定：1列佈局
- 發文歷史：水平滾動表格
- 互動分析：全寬圖表 + 1列統計卡片

## 交互設計

### 1. 導航
- **返回按鈕**: 返回 KOL 列表頁面
- **刷新按鈕**: 重新載入數據
- **編輯按鈕**: 進入編輯模式（需要權限）

### 2. 表格操作
- **查看詳情**: 點擊貼文 ID 查看完整內容
- **排序**: 點擊列標題進行排序
- **篩選**: 使用搜索框篩選貼文
- **分頁**: 支援分頁瀏覽

### 3. 圖表交互
- **時間範圍選擇**: 選擇不同的時間範圍
- **數據點懸停**: 顯示詳細數據
- **縮放**: 支援圖表縮放

### 4. 數據導出
- **CSV 導出**: 導出發文歷史數據
- **Excel 導出**: 導出完整 KOL 數據
- **PDF 報告**: 生成 KOL 表現報告

## 狀態管理

### 載入狀態
- 頁面載入時顯示骨架屏
- 數據載入時顯示載入動畫
- 錯誤時顯示錯誤訊息

### 編輯狀態
- 進入編輯模式時，可編輯欄位變為輸入框
- 保存按鈕變為可用狀態
- 取消按鈕恢復原始數據

### 數據更新
- 即時更新：從 Google Sheets 即時讀取
- 手動刷新：點擊刷新按鈕
- 自動刷新：可設定自動刷新間隔

## 性能優化

### 1. 數據載入
- 分頁載入發文歷史
- 虛擬滾動處理大量數據
- 緩存常用數據

### 2. 圖表渲染
- 使用 Canvas 渲染大量數據點
- 按需載入圖表數據
- 防抖處理用戶交互

### 3. 組件優化
- 使用 React.memo 避免不必要的重渲染
- 使用 useMemo 和 useCallback 優化計算
- 懶載入非關鍵組件

## 無障礙設計

### 1. 鍵盤導航
- 支援 Tab 鍵導航
- 支援 Enter 鍵確認操作
- 支援 Esc 鍵取消操作

### 2. 螢幕閱讀器
- 提供適當的 ARIA 標籤
- 描述圖表和表格內容
- 提供操作反饋

### 3. 視覺設計
- 足夠的顏色對比度
- 清晰的字體大小
- 適當的間距和佈局
